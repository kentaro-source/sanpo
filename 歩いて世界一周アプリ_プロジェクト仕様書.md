# 歩いて世界一周アプリ プロジェクト仕様書

## プロジェクト概要

スマホの歩数計と連動して、歩くだけで世界193カ国の首都を一筆書きで巡る世界一周をシミュレーションするアプリ。訪問済み国に到達したら過去の思い出が表示される。自分用。

---

## 背景・目的

- 既存の「散歩で日本一周」アプリの世界版がほしい
- Travel Trackerで訪問国データを持っているので連携して楽しみたい
- 日常の散歩がエンターテイメント化する
- 健康管理と旅の思い出の可視化を同時に実現

---

## 機能要件

### 1. ルート設計

#### 対象国
国連加盟国193カ国の首都を経由地とする。

#### ルート決定方法
地理的に近い順で一筆書きとなるルートをあらかじめ設計。
例：東京→ソウル→平壌→北京→ウランバートル→モスクワ→…→ロンドン→パリ→…→ブエノスアイレス→…→ワシントンD.C.→…→東京

#### 初期ルートの生成方針
- 日本（東京）をスタート地点
- アジア → 中東 → アフリカ → ヨーロッパ → 南北アメリカ → オセアニア → 日本に戻る流れ
- 各区間は首都間の直線距離（大圏距離）で計算
- 海を渡る区間は「船旅」として扱う（海峡やトンネル、空路などは気にしない、ロマン優先）

#### ルートデータの管理
- 193カ国の首都の緯度経度とルート順を定義したJSONファイルを持つ
- 首都間を複数のマスで区切る（桃鉄のマス目の概念）
- マス数は首都間の距離に応じて可変（例：近い首都間は3〜5マス、遠い首都間は10〜20マス）
- ルートは後から調整可能（JSONを編集するだけ）

### 2. 歩数→サイコロ連動

#### 歩数からサイコロを獲得
- スマホの歩数計APIから1日の歩数を取得
- 一定歩数ごとにサイコロ1個獲得（デフォルト：5,000歩=1個、設定で変更可能）
- 獲得したサイコロは貯めておける
- 忙しくてアプリを開けない日があっても、後で一気に振って進められる

#### サイコロを振る
- ユーザーが手動でボタンを押して振る（桃鉄スタイル）
- サイコロは1〜6の目が出る
- 出た目の数だけマスを進む
- 1日に複数回振れる（貯めたサイコロの数だけ）

#### 進行表示
- 地図上で現在位置をピンで表示
- 通過済みのマスと未通過のマスを色分け
- 次の首都までの残りマス数を表示
- 残りサイコロ数を常に表示

### 3. マスのイベント

各マスを踏むとランダムでイベントが発生する。マスの種類はエリア（大陸・地域）に応じた内容。

#### マスの種類（初期実装）
- 🏙️ **都市マス**：経由地の都市情報を表示
- 🏔️ **観光マス**：その地域の世界遺産や名所情報
- 🍜 **グルメマス**：その地域の名物料理
- ✈️ **空港マス**：Travel Trackerの搭乗履歴があればエアライン情報を表示
- 🎰 **カジノマス**：カジノがある国に設置。訪問歴があれば思い出
- 🎲 **イベントマス**：ランダムな豆知識、雑学

#### エリア別の味付け
- 中東ではラクダや砂漠のイメージ
- オセアニアではビーチやサーフィン
- 東欧では旧ソ連的な雰囲気
- 地域性が出ると旅情が増す

#### マスイベントのデータ
- エリア別のイベント素材をJSONで管理
- Claude APIは使わず、事前に用意したデータからランダム選択
- 後から素材を追加できる構造に

### 4. 首都到達時のイベント

#### 訪問済み国（Travel Trackerの訪問国リストと連動）
到達した国が訪問済みの場合：
- 「○○に到着！ここは△△年に訪れた国です」
- 訪問時の思い出メモや写真を表示（Travel Trackerのデータから）
- カジノ訪問歴があれば「カジノで遊んだ国」バッジ表示

#### 未訪問国
到達した国が未訪問の場合：
- 「○○に到着！まだ訪れたことのない国です」
- その国の基本情報（人口、言語、通貨、公用語等）を表示
- RestCountries APIで国情報を取得

#### 共通表示
- 国旗
- 首都名と国名
- 次の目的地と残りマス数
- 到達日時を記録

### 5. 地図表示

- 世界地図上に現在地を表示
- 通過済みのマスと未通過のマスを色分け
- 訪問済み国はTravel Trackerデータに基づいて地図上でハイライト
- マスをタップするとマス情報のポップアップ
- 首都ピンをタップすると首都情報のポップアップ

### 6. 記録と統計

- 今日の歩数・獲得サイコロ数
- 累計歩数・累計マス数
- 通過した首都の数
- スタートからの経過日数
- 月別の歩数カレンダー（既存の散歩アプリと同様）

### 7. ストリートビュー（オプション）

- 現在の地図上の位置でGoogle Street Viewを開ける
- 「今ここを歩いています」の景色を確認できる
- ※Google Maps API使用（無料枠あり）

### 8. 完走後

完走後の挙動は未定。まずは無限にリセットされて同じルートを回る形で実装し、完走したときに改めて考える。

- 完走したら「世界一周達成！」の演出
- 累計周回数をカウント
- 2周目以降の挙動は後日決める

## 将来の拡張（桃鉄化）

シンプル版が完成して遊んでみた上で、以下の要素を追加する可能性あり。今の段階では実装しないが、データ構造は将来の追加を意識して設計しておく。

### 桃鉄要素の候補
- **物件購入**：首都や観光地に物件を配置し、購入可能に
- **収益システム**：周回のたびに自分の物件から収益
- **AI対戦**：AI複数人と対戦形式で世界一周
- **貧乏神・ボンビー的な妨害要素**
- **独占ボーナス**：同じ国の物件を全部買ったらボーナス
- **カード**：桃鉄の急行カードやダビング屋のような特殊能力

### 設計上の配慮
- マスデータは将来「物件情報」を追加できる構造にしておく
- プレイヤーデータは将来「所持金」「所有物件リスト」を追加できる構造にしておく
- ゲームロジックは独立したモジュールにして、後からAI対戦を追加できるように

## 技術構成

| 要素 | 技術 |
|------|------|
| フロントエンド | React（PWA対応、スマホファースト） |
| 歩数取得 | Web APIまたはHealthKit/Google Fit |
| 地図 | Leaflet + OpenStreetMap（無料）またはGoogle Maps |
| ストリートビュー | Google Maps API |
| 国情報 | RestCountries API |
| 訪問国データ | Travel TrackerのSupabaseから取得 |
| ルートデータ | ローカルJSON |
| データ保存 | Supabase（個人用、Travel Trackerと共通） |
| ホスティング | Cloudflare Pages / Vercel / Netlify（無料枠） |

## 開発の進め方

1. **Phase 1（MVP）**: ルート＋マスデータ作成（193首都）＋地図表示＋歩数→サイコロ連動＋サイコロを振って進む
2. **Phase 2**: マスイベント（エリア別のランダムイベント）
3. **Phase 3**: 首都到達イベント（訪問済み/未訪問の出し分け）、Travel Tracker連携
4. **Phase 4**: ストリートビュー連携、記録統計
5. **Phase 5**: 完走後の挙動（やりながら決める）
6. **Phase 6（将来）**: 桃鉄要素の追加（物件、AI対戦等）

Phase 1から始めて、歩数でサイコロを振ってピンが進むところまで最初に作る。

---

## 注意事項

- サイコロ獲得レートは最初は「5,000歩=1個」で始めて、進みが遅すぎたら調整する
- 首都間のマス数は距離に応じて可変（近い首都間3〜5マス、遠い首都間10〜20マス程度）
- 193カ国全部回るのに数ヶ月〜1年程度を想定（調整可能）
- 193首都のルートデータ＋マスデータは手動整備が必要（Claude Codeに生成させる）
- Travel Trackerとデータベースを共有するため、個人用Supabaseプロジェクトを使用
- 完走後の設計は「やりながら考える」で問題なし
- 桃鉄要素は将来の拡張として、データ構造は拡張しやすい形にしておく
